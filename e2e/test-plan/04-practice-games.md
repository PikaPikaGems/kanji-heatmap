# 04 — Practice Games (P2)

Guards the three practice screens against refactors of deck building
(`src/components/shared-practice/`), game state machines, and the
activity-recording pipeline (`src/lib/activity/storage.ts`).

**Already covered** (`e2e/practice.spec.ts` — don't duplicate):
- Reading practice: start button → game screen shows kana input
- Writing practice: initial screen renders
- Speed Katakana: correct romaji advances word 1 → 2 (パーセント → アメリカ)

**Conventions:** import `test`/`expect` from `../fixtures` (network
stubbed); 30s timeouts on anything gated by worker init or JSON loading;
assert via roles/labels/visible text, never CSS classes or component
internals.

---

## Reading practice (`/reading-practice`)

### RP-1: Correct answer advances progress
- **Guards:** answer-checking + progress logic in
  `src/components/recognition-practice-v1/`.
- **Steps:**
  1. `goto /reading-practice`, click "Start Practicing"
  2. Determine the prompted kanji's expected reading — deterministic
     seeding needed: fix the deck via the settings drawer before
     starting, or read the expected answer from the visible hint/reveal
     UI
  3. Type the correct kana into the textbox, submit (Enter)
- **Assertions:** progress indicator increments; input clears; next card
  is a different kanji.
- **Note:** if deck order is random, seed determinism via
  URL/localStorage settings (`recognition-practice-v1-settings`) rather
  than changing product code.

### RP-2: Wrong answer gives feedback without advancing
- **Steps:** start a session, type an obviously wrong reading, submit.
- **Assertions:** feedback state shown (error/reveal); progress count
  unchanged.

### RP-3: Session completion records activity
- **Guards:** activity write path (`activity-all-time`,
  `activity-by-day`).
- **Steps:** configure the smallest possible session in the settings
  drawer, answer all cards (or use skip/reveal if available).
- **Assertions:** completion screen visible; then
  `page.evaluate(() => localStorage.getItem("activity-all-time"))` is
  non-null; `goto /dashboard` shows non-empty stats instead of
  "Saved only on this device".

## Speed Katakana (`/speed-katakana`)

Deterministic under default settings: word order fixed, set 1 starts
パーセント (1 / 48). Matching engine: `src/lib/speed-katakana-match.ts`.
Challenge data is generated by `scripts/generate-speed-katakana.mjs`
(already run by the Playwright webServer command).

### SK-1: "skip" advances without credit
- **Steps:** Start Game, type `skip` in the answer box (placeholder:
  `Type romaji or "skip"`).
- **Assertions:** advances to 2 / 48; input cleared.

### SK-2: Wrong romaji does not advance
- **Steps:** Start Game, type `banana`.
- **Assertions:** still 1 / 48; パーセント still visible.

### SK-3: Challenge-set selector switches sets
- **Guards:** `ChallengeSetSelector.tsx` + generated
  `public/json/katakana/challenge-set-<N>.json` loading.
- **Steps:** on the initial screen pick set 2, start.
- **Assertions:** word counter shows set-2 length; first word differs
  from パーセント.

### SK-4: Completing a set reaches EndSession and persists stats
- **Steps:** pick the smallest set (or drive all 48 words by computing
  each word's romaji in `page.evaluate` — feasible since order is
  fixed).
- **Assertions:** `EndSession` screen visible (results summary); per-set
  stats key written to localStorage; dashboard speed-katakana heatmap
  shows activity.
- **Priority note:** the full-set loop is slow (~30–60s); mark
  `test.slow()` and keep it the single long-runner in this file.
