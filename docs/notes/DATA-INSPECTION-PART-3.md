## Key Takeaways

1. **The two sources disagree on type for 994 kanji (41%)** — this is a fundamental disagreement in classification methodology, not just data quality. Neither source is definitively "correct" since kanji etymology is debated by scholars.

2. **Semantic agreement is high (92%)** when both sources have data. The 105 disagreements are mostly radical-vs-variant representation differences, not substantive disagreements.

3. **Phonetic agreement is decent (87%)** when both have data. Disagreements often reflect different decomposition depth (immediate compound vs core phonetic).

4. **kanjidict.txt can fill 381 "unknown" types** in structure.json — this is low-hanging fruit.

5. **494 keisei entries in structure.json have no phonetic in kanjidict** — because kanjidict classifies them as compound ideograph or doesn't have them at all. This is the core tension between the two sources.

6. **51 rare characters in structure.json don't exist in kanjidict.txt** — these are obscure components that only appear as phonetic/semantic elements in other kanji.

## Recommendation

Given the significant disagreements, consider:
- Using kanjidict.txt to fill the 381 "unknown" gaps in structure.json
- For type disagreements, prefer structure.json (hlorenzi source) for now since it was curated specifically for this project
- For semantic/phonetic data, structure.json is more detailed (it lists both components for keisei), while kanjidict only lists the primary radical
- Keep both sources available for cross-referencing when users/contributors spot errors
